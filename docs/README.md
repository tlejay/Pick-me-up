# PICK-ME-UP Documentation

## Directory Structure

### /api
API documentation, endpoint specifications, and integration guides

### /architecture
System architecture diagrams, technical design documents, and infrastructure designs

### /business
Business requirements, user stories, and product specifications

### /user-guides
End-user documentation, admin guides, and training materials

## Documentation Standards

- Use Markdown format for all documentation
- Include diagrams where applicable (Mermaid, PlantUML)
- Keep documentation up-to-date with code changes
- Version all major documentation changes
